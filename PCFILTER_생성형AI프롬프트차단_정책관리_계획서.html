1. 개요 (Context)

현재 생성형 AI 프롬프트 차단 기능은 ChatGPT 단일 기준으로 설계되어, 관리 정책 화면에서 생성형 AI 프롬프트 항목을 차단/허용 라디오(is_block) 하나로만 일괄 제어합니다.

AI 서비스가 빠르게 늘어남에 따라 프로그램별로 차단/허용을 다르게 설정할 수 있어야 합니다. 한 화면에 항목을 계속 추가하면 UI가 비대해지므로, 별도 정책 관리 탭에서 정책을 만들고, 관리 / 오프라인 / 개별 정책에서는 그 정책을 드롭다운으로 선택하는 구조로 전환합니다.

[정책 관리 탭] (신규) | "생성형 AI 프롬프트 차단 정책" 생성 | - 정책 기본 차단/허용 (정책당 1개) | - 생성형 AI별 제어 (차단 / 허용 / 기본값) | - 프로그램 목록/URL은 default_ai_t에서 조회 v [정책 저장] -> policy_t (policy_type = 'chatgpt_process_exe') | v [관리/오프라인/개별 정책] | is_block 라디오 제거 -> 위 정책을 드롭다운으로 선택 (chatgpt_pid 참조) v [에이전트 동기화] setAgentCashModify -> agent_t.cash=false | v [에이전트] 프로그램별 차단/허용 적용
의도하는 결과: 운영자가 "생성형 AI 프롬프트 차단 정책"을 자유롭게 만들고(프로그램별 차단/허용), 관리·오프라인·개별 정책에서 그 정책을 선택만 하면 되는 구조. 다른 정보유출방지 기능(출력물/매체)과 100% 동일한 UX.

2. 요구사항 요약

기능상세
신규 정책 관리 탭 화면을 2탭 구조로 전환. 생성형 AI 프롬프트 차단 관리 정책(기존) + 생성형 AI 프롬프트 차단 정책 관리(신규). 신규 탭에서 정책 목록 조회 / 추가 / 수정 / 삭제. 목록 컬럼: 순번 / 정책명 / 최근 수정일자.
신규 정책 추가/수정 팝업 ① 정책 기본 차단/허용(정책당 1개) ② 생성형 AI별 설정 탭 — 프로그램별 제어(차단/허용/기본값).
변경 관리 정책 생성형 AI 프롬프트 차단/허용 라디오 제거 → 정책 선택 드롭다운으로 교체.
신규 오프라인 정책 chatgpt에 현재 없는 오프라인 섹션 신규 추가. 오프라인 사용 토글 + 오프라인 정책 드롭다운.
변경 개별 정책 개별정책 추가/수정 모달의 add_is_block/view_is_block 라디오 → 정책 선택 드롭다운으로 교체.
유지 예외 정책 chatgpt_except는 차단/허용 개념이 없어(대상 제외만) 변경 없음.
변경 신규 라이선스 시딩 신규 라이선스 생성 시 chatgpt_process_exe 기본정책(pid 0) INSERT, 관리/오프라인 기본값이 pid 0을 참조하도록 정렬.

3. 설계 개요 및 네이밍 사전

3.1 핵심 설계 원칙

  • 매체/출력물 패턴 미러링: 드롭다운에서 선택되는 "이름 있는 정책" = *_process_exe 패턴을 그대로 차용 → chatgpt_process_exe 신설
  • 기존 chatgpt 접두어 유지: policy_type / 메서드명 / URL을 모두 chatgpt_* · Chatgpt 규칙으로 통일. 라이선스 키(chatgpt)·좌측 메뉴도 기존 그대로. UI 표기만 "생성형 AI"
  • 프로그램 목록 외부 의존: AI 프로그램 목록/URL은 default_ai_t(DBPatcher에서 생성·시딩)에서 읽기만. pf3-adm은 추가/삭제하지 않음. (URL 수정 기능은 추후 공통설정에서)
  • 정책당 기본 차단/허용 존재: 정책의 기본 is_block + 프로그램별 제어값은 ai_list 배열의 block 값으로 보관
  • 데이터 마이그레이션 경계: 기존 운영 라이선스의 is_block → chatgpt_process_exe 정책 변환은 DBPatcher(runQueryLicense)가 수행. pf3-adm은 변환된 pid 0(기본정책)/개별 정책을 가져다 사용
  • 에이전트 캐시 무효화 유지: 모든 정책 수정 경로에서 기존처럼 setAgentCashModify 호출

4.2 default_ai_t (신규, DBPatcher 생성·시딩 — pf3-adm은 읽기 전용)

// 프로그램 1행 = URL 여러 개. url 은 PostgreSQL 배열 (확정)
CREATE TABLE default_ai_t (
    pno   bigserial    NOT NULL,        // PK (시퀀스)
    name  character varying(32) NOT NULL, // 프로그램명 (ChatGPT, Claude, ...)
    url   text[],                          // 식별 URL 목록 (예: {"chat.openai.com","chatgpt.com"})
    CONSTRAINT default_ai_t_pkey PRIMARY KEY (pno)
);  // COMMENT ON TABLE default_ai_t IS 'AI 기본'
저장 방식장점판단
url text[] (배열)기존 policy_sub_t.sub_text TEXT[] 컨벤션과 일치. "한 줄에 하나" 입력 → \n split → 배열. 조회 array_to_string(url, E'\n'). 매칭 = ANY(url). 구분자 모호성 없음권장
url jsonb (배열)policy_t의 JSONB-everywhere 스타일과 일관. URL별 메타(사용여부 등) 확장 여지대안
단일 text + 구분자—비추천 파싱 모호/쿼리 불편
입력 UX는 이미지 6의 생성형 AI URL 설정 팝업 — "한 줄에 하나의 URL 입력(예: chatgpt.com/*)". 정보 아이콘: "등록된 URL은 생성형 AI 서비스 식별 기준이며, 해당 서비스에 개인정보 입력 시 차단이 적용됩니다."

DB.4 라이선스 마이그레이션 — is_block → chatgpt_process_exe (runQueryLicense)

기존 운영 라이선스의 chatgpt/chatgpt_separate 정책(단일 is_block)을 이름 있는 정책(chatgpt_process_exe)으로 변환하고, 원 정책은 chatgpt_pid로 참조 치환. chatgpt_pid IS NULL 가드로 멱등성 확보. 총 4단계.

#대상동작
①chatgpt (관리)is_block → chatgpt_process_exe 기본정책(pid 0) 생성. ROW_NUMBER()-1로 pid 0 부여
②chatgpt (관리)is_block 제거 → chatgpt_pid=0 + alarm + offline{use,policy{pid,chatgpt_pid,alarm}} 추가
③chatgpt_separate (개별)is_block → 개별 chatgpt_process_exe 정책 생성. max(pid)+ROW_NUMBER()로 기본정책 이후 pid 부여
④chatgpt_separate (개별)name 매칭으로 ③의 pid를 chatgpt_pid에 연결 + alarm + offline 추가
 
// ① 관리정책 is_block → 기본정책(pid 0) 생성
INSERT INTO policy_t (... pid ...)
SELECT ..., ROW_NUMBER() OVER (ORDER BY pid)-1 AS pid, ...
FROM ( SELECT license_code, 'dlp', 'chatgpt_process_exe', pid, pvn,
    json_build_object(
        'name', '기본정책',
        'is_block', (policy_json->>'is_block')::boolean,
        'ai_list', ( // default_ai_t 전체를 {name,url,block:1} 배열로
            SELECT json_agg(json_build_object(
                'name', d.name, 'url', array_to_json(d.url), 'block', 1
            ) ORDER BY d.pno) FROM default_ai_t d ),
        'policy_hash', ('chatgptnormalpoli' || to_char(now(),'YYMMDDHH24missms'))
    ), ...
  WHERE ... policy_type='chatgpt' AND policy_json->>'chatgpt_pid' IS NULL ) t;

// ② 관리정책 is_block 제거 → chatgpt_pid + alarm + offline
UPDATE policy_t SET policy_json = (policy_json::jsonb - 'is_block')
    || jsonb_build_object('chatgpt_pid', 0)
    || jsonb_build_object('alarm', false)
    || jsonb_build_object('offline', jsonb_build_object(
            'use', false,
            'policy', jsonb_build_object('pid',0,'chatgpt_pid',0,'alarm',false)))
WHERE ... policy_type='chatgpt' AND policy_json->>'chatgpt_pid' IS NULL;
