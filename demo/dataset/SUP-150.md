## 문의 요약
- 실명인증(본인확인) 요청이 반복적으로 실패한다는 문의.
- 조사 대상: `demo/logs/id-verify.log` 의 ERROR/WARN 정황 및 Jira 유사 사례.

## 원인 분석 (로그 근거 인용)
- 다수의 verify 요청이 동일 원인으로 실패하고 있음:
  - `09:14:22 ERROR [idv] verify FAILED rid=IDV-20260709-0007 provider=NICE code=E402 reason="provider auth token expired"`
  - `09:15:48 ERROR ... rid=IDV-20260709-0012 code=E402 reason="provider auth token expired"`
  - `09:18:31 ERROR ... rid=IDV-20260709-0021 code=E402 reason="provider auth token expired"`
- 결정적 근거 — 토큰 갱신 자체가 실패해 재발급 전까지 모든 호출이 실패함:
  - `09:22:36 ERROR [idv] provider token refresh FAILED — all verify calls will fail until apiKey is reissued`
- 실패 규모 및 부가 문제(개인정보 평문 적재):
  - `09:20:10 WARN [idv] 34 verify requests failed in last 10m (provider auth token expired) — PII(name/rrn/phone/email/ip) written to error log in cleartext`
- **1차 원인**: 본인확인 제공사(NICE) 인증 토큰이 만료(code=E402)되었고 토큰 자동 갱신(refresh)마저 실패하여, apiKey 재발급 전까지 모든 실명인증이 실패하는 상태.
- **부가 문제**: 실패 로그에 이름·주민번호·휴대폰·이메일·클라이언트 IP 등 개인정보(PII)가 평문으로 기록되고 있음 — 유출·컴플라이언스 리스크.

## 유사 사례 (Jira)
- **SUP-150 — 본인확인(실명인증) 연동 실패 및 오류 로그에 개인정보 평문 적재** (고객사 G, 2026-03-05)
  - 증상 일치: `verify FAILED ... code=E402 reason="provider auth token expired"` 반복 후 `provider token refresh FAILED http=401 (invalid_client: apiKey revoked)`.
  - 원인: 본인확인 제공사 API 키(apiKey/토큰)가 폐기·만료되어 재발급 전까지 모든 verify 호출 실패.
  - 해결: apiKey 재발급·교체로 verify 복구, 오류 로그 PII 마스킹(`mask_pii_in_log=true`), 토큰 만료 30일 전 알림 및 자동 갱신 추가로 재발 방지.

## 해결 방법
1. **제공사(NICE) API 키(apiKey) 재발급·교체**: 폐기·만료된 키를 새 키로 교체하고 토큰 갱신을 정상화 → verify 복구.
2. 교체 후 실명인증 정상 동작(신규 verify 성공, `code=E402`/token refresh 오류 소멸)을 확인.
3. **개인정보 평문 로깅 차단**: 로깅 정책을 `mask_pii_in_log=true` 로 변경해 이름/주민번호/휴대폰/이메일/IP 마스킹. 이미 남은 평문 PII 로그는 정책에 따라 안전하게 정리.

## 권장 후속 조치
- **토큰 만료 사전 알림**(예: 만료 30일 전)과 **자동 갱신** 구성으로 동일 장애 재발 방지.
- 토큰 갱신 실패에 대한 모니터링/알림을 추가해 `token refresh FAILED` 발생 즉시 담당자에게 통지.
- PII 로깅 정책을 전사 로그로 점검·표준화하여 다른 모듈에서도 평문 PII 적재가 없는지 확인.
