## 문의 요약
이미 승인 완료된 파일을 동일하게 다시 업로드했을 때, 관리자페이지에서 재승인이 처리되지 않는다는 문의입니다.

## 원인 분석 (로그 근거 인용)
- `demo/logs/dlp-approve-api.log` 에서 재승인 시도 흐름이 확인됩니다. 승인 확인(check)·요청(POST)은 모두 정상 응답(200)으로 기록됩니다.
  - `"PUT /client/v1/dlp/approve/check/CUST9931/2438/fileattach HTTP/1.1" 200`
  - `"POST /client/v1/dlp/approve/CUST9931/2438/fileattach HTTP/1.1" 200`
- 그러나 최초 승인으로 문서가 이미 완료 상태가 된 뒤 재요청 시, 갱신이 무시됩니다.
  - `2026-07-08 15:02:33 INFO [adm] approve changed doc=2438 ... result=완료`
  - `2026-07-08 15:57:45 WARN [adm] approve update ignored doc=2438 reason=already_completed(approve_status=1)`
- 즉 승인 상태가 이미 '완료(approve_status=1)'인 문서는 신규 승인 데이터가 생성되지 않아 재승인이 반영되지 않습니다. 실제로 임시 조치로 approve_no 를 초기화한 뒤에야 재승인이 정상 처리되었습니다.
  - `2026-07-08 17:06:00 INFO [adm] tmp-fix approve_no reset doc=2438 by=[EMAIL_2]`
  - `2026-07-08 17:13:47 INFO [adm] approve changed doc=2438 result=완료 (after reset)`
- 참고: `demo/logs/admin-approve.log` 에는 의견서 저장 시 별개의 DB 오류(`PSQLException: unterminated quoted string`, doc=2439/2441)가 있으나, 이는 이번 재승인 미반영 증상(doc=2438)과는 직접 관련이 없는 별도 이슈로 보입니다.

## 유사 사례 (Jira)
- **SUP-141** (고객사 B, 2025-12-03): "이미 승인 완료된 결재를 재요청 시 재승인이 반영되지 않음". 동일 파일(해시 동일)로 재요청 시 관리서버가 '이미 완료된 건'으로 판단해 상태를 갱신하지 못함. `approve check/request`는 200이지만 `approve update ignored ... already_completed`, `re-approval requested but not reflected` 로그가 남는 증상으로, 이번 문의와 동일한 패턴입니다.
- 원인: **승인 갱신 쿼리 조건 누락**으로 완료 상태에서도 신규 승인 데이터가 생성되지 않음.

## 해결 방법
- **근본 수정**: 승인 상태가 '대기'일 때만 기존 데이터를 갱신하고, 이미 '완료'된 경우에는 신규 승인 데이터가 생성되도록 승인 갱신 로직을 수정합니다 (SUP-141 해결책과 동일).
- **긴급 임시 조치**: 재승인이 필요한 문서의 `approve_no` 값을 초기화하면 재요청 시 재승인이 정상 생성됩니다. 로그상 doc=2438 에서 이 조치 후 정상 처리(`approve changed ... after reset`)가 확인되었습니다.

## 권장 후속 조치
1. SUP-141 수정 패치의 현재 배포 여부를 확인하고, 미적용 시 해당 로직 수정본을 반영하세요.
2. 급한 재승인 건은 approve_no 초기화로 우선 처리하되, 임시 조치는 근본 패치 적용 전까지의 한시적 대응으로만 사용하세요.
3. `already_completed` 로 무시되는 재승인 시도에 대해 사용자에게 명확한 안내 메시지가 노출되도록 UX/에러 핸들링 보완을 검토하세요.
4. 별도로 관측된 의견서 저장 시 `PSQLException: unterminated quoted string` (doc=2439/2441) 오류는 입력값에 포함된 작은따옴표(')의 SQL 이스케이프/파라미터 바인딩 문제로 의심되므로, 별건으로 추가 조사 및 파라미터라이즈드 쿼리 적용을 권장합니다.
