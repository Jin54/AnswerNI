# AnswerNI 데모 데이터셋 — 문의 / 진행상황 / 최종보고서 (SUP-140~149)

`문의샘플.txt`의 10건을 **실제 파이프라인(`app.agent.run_agent`)에 그대로 돌려** 얻은 결과 데이터셋입니다.
각 파일은 실제 실행에서 나온 진행 트레이스·Jira 매치·최종 보고서를 담습니다(합성 데이터, 재실행 시 값이 달라질 수 있음).

- **실행 백엔드**: 로컬 데몬(도구 실행·PII 마스킹·SLM 요약) + 원격 LLM(Claude Code CLI, `claude-opus-4-8`). `.env`에 `ANTHROPIC_API_KEY`가 없어 CLI 폴백 경로로 구동됨.
- **각 파일 구성**
  - `## 문의` — 고객 문의 원문 (`문의샘플.txt`)
  - `## 진행상황 (실제 실행 트레이스)` — `run_agent`가 emit한 실제 이벤트: 원격 LLM 판단, 도구 호출(막다른 검색 포함), 결과 요약, PII 마스킹(원본→치환), SLM 압축, Jira 결과 수신, 종료(stop_reason·누적 도구 호출 수)
  - `## 유사 Jira 사례` — 도구가 실제로 회수한 매치(`demo/jira/issues.json` 기반)
  - `## 최종보고서` — 원격 LLM이 생성한 `# 기술지원 보고서` 전문(문의 요약 / 원인 분석(로그 근거 인용) / 유사 사례(Jira) / 해결 방법 / 권장 후속 조치)
- **PII 마스킹**: 로그의 이메일·휴대폰·주민번호·IP는 로컬 에이전트가 `[EMAIL_n]`·`[PHONE_n]`·`[RRN_n]`·`[IP_n]` 토큰으로 치환(케이스별 1번부터). 원문 PII는 원격 LLM에 전달되지 않음.
- **재생성 방법**: `python <scratchpad>/gen_dataset.py [SUP-140 ...]` (인자 없으면 10건 전체). 프로젝트 루트에서 실행.
