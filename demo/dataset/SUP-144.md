## 문의 요약
고객사에서 에이전트 알림 팝업의 노출 시간을 조절하고 싶다는 개선 문의. 현재 조절 가능 여부와 대응안을 정리 요청.

## 원인 분석 (로그 근거 인용)
- system.log 에서 `popup`, `notification` 키워드로 조회했으나 관련 줄이 없어, 이번 문의는 장애가 아닌 **기능 사양(하드코딩된 고정값)** 이슈로 판단된다.
- system.log 에 확인된 오류/경고는 이번 문의와 무관한 별개 사안이다:
  - `2026-07-08 03:41:10 ERROR [sys] write failed: No space left on device (/var/log/app.log)`
  - `2026-07-08 03:41:10 ERROR [app] failed to append audit log: No space left on device`
  - `2026-07-08 03:45:00 ERROR [api] 503 Service Unavailable (audit write failed)`
  - `2026-07-08 02:14:52 WARN [sys] disk usage /var/log at 92%` / `03:42:33 ... at 100%`
  → 이는 /var/log 디스크 100% 포화로 인한 감사로그 기록 실패 건으로, 팝업 시간 조절과는 관련이 없다. (별도 조치 필요, 아래 후속 조치 참고)

## 유사 사례 (Jira)
- **SUP-144 — 에이전트 알림 팝업 노출 시간 조절 기능 요청 (개선 문의)** / 고객사 D(의료) / 2026-05-22
  - 내용: Mac 에이전트의 차단·안내 알림 팝업 노출 시간이 너무 짧거나 길다며 관리자 정책으로 조절 가능하게 해달라는 요청. **현재는 노출 시간이 고정값으로 하드코딩**되어 있음.
  - 처리: 관리자페이지 정책에 **'알림 팝업 노출 시간(초)' 설정 항목 추가** 및 에이전트가 정책값을 반영하도록 개선 예정. **로드맵 반영, 다음 마이너 버전 목표.** 임시 안내: 현재 버전은 고정값이라 조절 불가.
- (참고) SUP-131 모바일 푸시 알림 미수신, SUP-140 결재 의견 작은따옴표 500 오류 — 키워드는 겹치나 이번 문의(팝업 노출 시간)와는 무관.

## 해결 방법
- **현재 버전에서는 조절 불가.** 알림 팝업 노출 시간이 코드상 고정값으로 하드코딩되어 있어 관리자 정책이나 설정으로 변경할 수 없다 (SUP-144 확인).
- 정식 대응은 **관리자페이지에 '알림 팝업 노출 시간(초)' 정책 항목이 추가되는 다음 마이너 버전** 적용이다. 해당 버전 배포 후 에이전트가 정책값을 반영하게 된다.

## 권장 후속 조치
1. 고객사에 **현재 조절 불가 + 다음 마이너 버전에서 정책 기반 조절 지원 예정(SUP-144 로드맵 반영)** 임을 안내.
2. 고객이 원하는 목표 노출 시간(초)과 대상 OS(Mac/Windows) 요구사항을 수집해 SUP-144 에 코멘트로 추가, 개발 반영 시 참고하도록 연계.
3. 버전 릴리스 일정 확정 시 고객사에 별도 통지. 급한 경우 개별 빌드 가능 여부를 개발팀에 확인.
4. (무관하지만 시급) system.log 에서 확인된 **/var/log 디스크 100% 포화 및 감사로그 기록 실패(503)** 는 별건으로 즉시 조치 필요 — 로그 정리/로테이션 및 디스크 증설 검토를 병행 티켓으로 진행 권고.
