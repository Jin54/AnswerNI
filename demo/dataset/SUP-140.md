## 문의 요약
결재(승인) 의견을 저장할 때 'Internal Server Error'(HTTP 500)가 발생하여 의견이 저장되지 않는다는 문의.

## 원인 분석 (로그 근거 인용)
`demo/logs/admin-approve.log` 에서 의견 저장 시점에 500 오류와 DB 예외가 반복 기록됨:

- `2026-07-09 14:21:03 ERROR [adm] 500 Internal Server Error while saving opinion doc=2439`
- `2026-07-09 14:21:03 ERROR [adm] org.postgresql.util.PSQLException: unterminated quoted string at or near "'"`
- `2026-07-09 14:25:12 ERROR [adm] 500 Internal Server Error while saving opinion doc=2441`
- `2026-07-09 14:25:12 ERROR [adm] PSQLException: unterminated quoted string at or near "'"`

`unterminated quoted string`(종료되지 않은 인용 문자열) 예외는, 의견 텍스트에 포함된 **작은따옴표(')가 SQL 쿼리의 문자열 경계로 오인**되어 쿼리 구문이 깨지면서 발생하는 전형적인 증상이다. 즉 사용자 입력값을 문자열로 직접 이어 붙여 쿼리를 구성(문자열 연결)하는 과정에서, 의견에 `'`가 1개 이상 들어가면 쿼리가 파손되어 500 오류로 이어진다.

참고로 `demo/logs/dlp-approve-api.log` 에는 ERROR 레벨 로그가 없어, 문제 지점은 결재 의견을 저장하는 admin-approve 처리 경로임을 확인했다.

## 유사 사례 (Jira)
동일 증상의 과거 이슈가 확인됨:

- **SUP-140** — "결재 의견에 작은따옴표(') 입력 시 500 Internal Server Error" (고객사 A, 2026-04-10)
  - 증상: DLP 결재 반려 의견에 `'`가 포함된 문장을 입력 후 저장하면 'Internal Server Error' 알림창이 뜨고 의견이 저장되지 않음. admin-approve 로그에 `unterminated quoted string at or near` 및 500 오류가 반복 기록됨.
  - 원인: 의견 문자열의 `'`가 SQL/쿼리에서 문자열 경계로 오인되어 쿼리가 깨짐.
  - 조치(resolution): 입력 의견의 작은따옴표를 이스케이프(`'`→`''`)하여 저장하도록 수정하고, **서버측 파라미터 바인딩(Prepared Statement)** 을 병행 적용해 근본 차단. 수정 후 정상 저장 확인.

이번 문의는 로그 증상(`unterminated quoted string`, 500)과 재현 조건(의견에 `'` 포함)이 SUP-140과 완전히 일치한다.

## 해결 방법
1. **근본 해결 — 파라미터 바인딩(Prepared Statement) 적용**: 의견 저장 쿼리에서 입력값을 문자열로 직접 연결하지 말고 바인딩 파라미터(`?` / placeholder)로 전달하도록 수정한다. 이렇게 하면 `'` 등 특수문자가 데이터로만 처리되어 쿼리 파손 및 SQL 인젝션이 원천 차단된다.
2. **즉시 완화 — 작은따옴표 이스케이프**: 코드 수정 배포 전 임시 대응으로, 저장 전 의견 문자열의 `'`를 `''`로 이스케이프 처리한다. (SUP-140에서도 병행 적용)
3. 수정 후 의견 텍스트에 `'`가 1개 이상 포함된 케이스로 재현 테스트하여 정상 저장을 확인한다.

## 권장 후속 조치
- 결재/의견 저장 외에도 사용자 입력을 쿼리에 직접 연결하는 다른 API 경로가 있는지 코드 점검(동일 패턴의 잠재 결함 및 SQL 인젝션 위험 제거).
- 회귀 방지를 위해 `'`, `"`, 개행 등 특수문자를 포함한 입력에 대한 저장 테스트 케이스를 추가.
- 배포 후 `demo/logs/admin-approve.log` 에서 `unterminated quoted string` / 500 오류가 재발하지 않는지 모니터링.
- 동일 증상 문의 시 참조할 수 있도록 본 건을 SUP-140과 연결(linked issue)하여 기록.
